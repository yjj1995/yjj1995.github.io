    ceph 集群的访问方式

    发布时间 : 2023-08-25 06:48
    阅读 :
    1. librados的访问
    2. 编程语言通过librbd和linux内核，librados 与集群通信
    3. 集群和客户端

    librados的访问

    一个 Ceph client 通过 librados 存放或者读取数据块到 Ceph 中，需要经过以下步骤：

    Client 调用 librados 连接到 Ceph monitor，获取 Cluster map。
    当 client 要读或者写数据时，它创建一个与 pool 的 I/O Context。该 pool 关联有 ruleset，它定义了数据在 Ceph 存储集群中是怎么存放的。
    client 通过 I/O Context 提供 object name 给 librados，它使用该 object name 和 cluster map 计算出 PG 和 OSD 来定位到数据的位置。
    client 直接和 OSD Deamon 交互来读或者写数据。

    编程语言通过librbd和linux内核，librados 与集群通信

    • 以go 语言为例
      • go语言的librbd 的源代码在 https://github.com/contiv/librbd，是使用 Go 语言实现的，调用 librados 的 C 库以及linux 内核模块，在 Ubuntu 14 上它的文件在 /usr/lib/x86_64-linux-gnu/librbd.so.
      • RBD 是 Rados Block Device 的缩写，而 librbd 是访问 RBD 的库。它调用 librados C 语言绑定，以及与Linux 内核中的 rbd 模块通信，来提供 RMD image 的创建、删除、映射和删除映射等操作。其中，创建和删除等操作是调用 librados，而将 RDB Image 映射到主机上则是调用 Linux rbd 内核模块。

    集群和客户端

    • Ceph 客户端和 Ceph 集群是密切相关的，它们之间的关系涉及到数据的访问和交互。

      • Ceph 集群：Ceph 集群是一个由多个节点组成的分布式存储系统。它包括存储数据的 OSD（Object Storage Daemon），管理存储池的 Monitor（MON），提供元数据服务的 MDS（Metadata Server），以及可能的对象存储网关和块设备服务。Ceph 集群负责存储、管理和复制数据，以确保高可用性、冗余和性能。

      • Ceph 客户端：Ceph 客户端是位于集群外部的系统，它通过不同的方式与 Ceph 集群进行交互。客户端可以是应用程序、服务器、虚拟机等，它们需要访问 Ceph 集群中存储的数据。Ceph 客户端使用不同的工具和接口来访问集群中的不同存储服务，如 RADOS、RBD 和 Ceph 文件系统。

    关系解释：

    • 当 Ceph 客户端想要访问存储在 Ceph 集群中的对象（Object）时，它使用 RADOS 客户端工具（如 rados 命令行工具或编程语言的 RADOS API）与 RADOS 存储集群交互。
    • 如果客户端想要访问 Ceph 集群中的块设备（RBD），它可以使用 RBD 客户端工具（如 rbd 命令行工具或编程语言的 RBD API）。
    • 如果客户端想要访问 Ceph 文件系统（CephFS）中的文件，它可以使用 CephFS 客户端工具（如 ceph-fuse 或 CephFS API）。
    • 总之，Ceph 客户端是外部系统与 Ceph 集群之间的桥梁，使应用程序能够访问和操作存储在 Ceph 集群中的各种数据。这些客户端工具和接口提供了各种方式来与 Ceph 集群交互，根据不同的用例和需求进行数据访问。
