    虚拟化

    发布时间 : 2023-08-08 01:38
    阅读 :
    1. 虚拟化是什么
    2. 虚拟化架构
    3. 分类
    4. 虚拟机监控程序
    5. 网络虚拟化
      1. 桥接
  • 处于同一个网段。- 交换机 - 本质是一个多端口的网桥，在数据链路层使用MAC 地址发送数据。 - 通过引入路由功能，一些交换机也可以在网络层转发数据（IP）。 - 交换机对数据包的转发是创建在MAC地址——物理地址之上的，对于IP网络协议来说，它是透明的。交换机主要关注于处理数据帧的物理地址（MAC地址），而不需要深入理解或处理数据帧中的IP地址。

    • 虚拟化是什么

    • 虚拟化技术的本质: 分区、隔离、封装和相对于硬件独立。
    • 任何一种把资源抽象成另一种形式的技术都是虚拟化技术。

    虚拟化架构

    • 宿主虚拟化

      • 在操作系统之上，装一个Hypervisor （虚拟机监控程序），通过监控程序创建子操作系统。

      • 调用硬件的流程是 虚拟机内核 -> Hypervisor -> 操作系统 -> 硬件，但这也导致了宿主虚拟化的性能是要比裸机虚拟化差很多的

      • 优点：
        1、硬件兼容性好
        2、使用方便，附加功能比较丰富

      • 缺点：
        1、性能差

    • 裸金属虚拟化

      • 不需要 底层有个 操作系统， 因为 虚拟监控程序本身就是一个操作系统

        • 例如：VMware ESXI 、Proxmox VE、Microsoft Hyper-V， 裸机虚拟化的性能是要优于宿主虚拟化的，一般IDC都是运行的这类系统

      • 裸型机在虚拟化中，虚拟监控程序直接访问硬件资源，不需要底层操作系统，可以将 虚拟监控程序 看成一个 简单的操作系统。

        • 调用的流程是 虚拟机内核 -> Hypervisor -> 硬件

      • 优点：

      • 1、性能好，损耗极低

      • 2、一般拥有资源管理、高可用性和安全方面的功能，比较适合企业数据中心

      • 缺点：
        1、硬件支持有限，因为hypervisor通常内置的设备驱动很有限，比如esxi

    分类


    虚拟机监控程序

    • Hypervisor （虚拟机监控程序）
    • 虚拟机监控程序是一种创建并运行虚拟机的软件，虚拟机监控程序（VMM） 会将虚拟机监控程序的操作系统和虚拟机的资源分离开，并创建和管理这些VM。

    网络虚拟化

    • 桥接（Bridge Adapter）
    • NAT
    • 主机
    • 内部网络

    桥接

    • 使用 虚拟交换机， 将虚拟机和物理机连接起来。

    • 处于同一个网段。
      - 交换机
      - 本质是一个多端口的网桥，在数据链路层使用MAC 地址发送数据。
      - 通过引入路由功能，一些交换机也可以在网络层转发数据（IP）。
      - 交换机对数据包的转发是创建在MAC地址——物理地址之上的，对于IP网络协议来说，它是透明的。交换机主要关注于处理数据帧的物理地址（MAC地址），而不需要深入理解或处理数据帧中的IP地址。
